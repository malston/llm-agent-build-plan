HOW TO READ THIS
This is not a curriculum. You don't need to learn the domain — you need reps with feedback on the parts most people skip. Each phase ends in something shippable; the proof-of-work doubles as your pivot evidence.
It is deliberately not tutorial-following, cert-collecting, or re-learning fundamentals you've had for fifteen years. If a week feels like coursework, you're off the rails.
THE ONE MOVE
Make everything measurable first, then build hard things under measurement, then go below the API.

THE THROUGHLINE
Every phase leans on something you already have — Go, observability depth, TDD/XP instincts, building in public — and points at your one true gap only once you've got the measurement scaffolding to evaluate it honestly.
The output isn't four toy projects. It's a connected body of work where the eval harness judges the reliable agent judges the fine-tune.
Failure modes
Tutorial hell
If you're following along rather than building your own thing, stop. Timebox the research slice hard.
Skipping evals because they're unglamorous
They're the whole point. Do not let Phase 1 get compressed away.
Polishing one project forever
The Friday demo forces forward motion. Ship rough, measure, move.
Going dark
Keep the thin networking thread alive — three months of silence is expensive to restart.
Pretending the model project is bigger than it is
One narrow fine-tune done honestly beats an ambitious one abandoned.
Flex the window
8 WEEKS — COMPRESS
Merge Phase 0 into Phase 1; one fine-tune only in Phase 3; Phase 4 becomes a half-week.
16 WEEKS — STRETCH
Two extra weeks on Phase 2 (a second, harder agent) and two on Phase 3 (a distillation and a fine-tune, compared).
Default: ~12 weeks. The phases stay the same shape — only their depth flexes.
